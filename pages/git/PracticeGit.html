Git 実践編

ローカルレポジトリの作成

今回作成したレポジトリをcode space上のubuntuにクローンしましょう
githubのレポジトリページからクローンするレポジトリのssh urlを取得し、codespaceのterminalで以下のコマンドを実行してください

terminal上で実行するコマンド

git clone 自分のレポジトリのurl

これでローカルレポジトリをcodespace上に作成することができました!

ローカルレポジトリでの編集

ローカルレポジトリ上で何かしらの編集を行います
まずは編集を行うためのテキストファイルを作成します。

cd レポジトリ名
touch practice.txt

今回は「practice.txt」というテキストファイルを作成します
さらに作成したファイルに何かしらの編集を行うためコードエディタで編集を行います
今回はcode spaceでの編集を前提としているのでvscodeで編集を行います

code practice.txt

何かしら編集を行ったら保存しましょう

リモートレポジトリの確認

github上で現在cloneしているレポジトリを見てみましょう
まだ、なんの変化も起こっていないはずです

リモートレポジトリに変更の適応

変更の記録

git add .

ローカルリポジトリに作業を登録

git commit -m "コミットメッセージ"

ローカルリポジトリをリモートリポジトリにアップロード

git push

リモートレポジトリの確認

これまでの作業を正しく行うことができればpractice.txtが追加されているはずです
変更を今までの方法で追加していけばバージョン管理を効率的かつ簡単に行うことができます。

リモートレポジトリの変更をローカルレポジトリに適応

github上で先ほど作成したpractice.txtを編集しましょう
レポジトリ上のpractice.txtをクリックすることでpractice.txtを開き
ペンのマークから編集を行うことができます

code space上で以下のコマンドを行うことで変更を適応させることができます

git pull

ローカルレポジトリの変更を確認

code practice.txt

うまくいっていれば変更が適応されているはずです
