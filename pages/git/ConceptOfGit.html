Git 概念編

Gitとは

分散型のバージョン管理システム
チーム開発や複数のバージョン管理に便利
WSLなどのCLI上での開発には必須といえるぐらいの効力を発揮

主に使う概念

gitを説明する内容はインターネット上にいくらでもあるので詳しい説明は省く
ここではgitを使う上で最低限の知識のみを記す

repository

コードや資料などをため込むことのできる貯蔵庫的なイメージ
ローカルレポジトリとリモートレポジトリがある。
作業を行うのは主にローカルレポジトリ

clone

リモートレポジトリの内容をローカル環境に複製し、ローカルレポジトリを作成する

branch

作業の流れを表す枝のような概念
複数人で同じ開発を行うと枝分かれをする。

add

自分のローカル環境での変更を記録する

commit

ローカルリポジトリに作業を登録

push

ローカルリポジトリをリモートリポジトリにアップロード

pull

リモートリポジトリからローカルリポジトリにダウンロード
